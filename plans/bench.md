Guide: API Benchmarking and Graphical Analysis with k6
This comprehensive guide walks through the complete lifecycle of setting up a REST API performance test suite using k6, running localized or automated benchmarks, and generating visual graphs to evaluate latency, throughput, and error rates.
1. Test Environment Setup
Ensure that the native k6 binary is installed on your local control plane or target runner. You can verify your installation by executing the version check command in your terminal shell.
# Verify k6 installation
k6 version
2. Crafting the Benchmarking Script
Create a test configuration file named api_bench.js. This script mimics actual user behavior by generating traffic concurrently using Virtual Users (VUs) over a fixed duration, while implementing verification checks to catch failed responses.
import http from 'k6/http';
import { check, sleep } from 'k6';

// Configure test parameters
export const options = {
  stages: [
    { duration: '30s', target: 20 }, // Ramp up to 20 users over 30s
    { duration: '1m', target: 20 },  // Stay at 20 users for 1 minute
    { duration: '10s', target: 0 },  // Ramp down to 0 users
  ],
  thresholds: {
    http_req_failed: ['rate < 0.01'], // Fail test if error rate is greater than 1%
    http_req_duration: ['p(95) < 250'], // 95% of requests must complete under 250ms
  },
};

export default function () {
  // Define target endpoint (Replace with your specific REST API URL)
  const url = 'https://httpbin.org/get';

  const params = {
    headers: {
      'Content-Type': 'application/json',
      'Authorization': 'Bearer YOUR_MOCK_TOKEN_HERE'
    },
  };

  const response = http.get(url, params);

  // Verify response payload and status integrity
  check(response, {
    'status is 200': (r) => r.status === 200,
    'transaction verified': (r) => r.body.includes('url'),
  });

  // Maintain a brief pacing interval between iterations
  sleep(1);
}
3. Generating Real-Time Visual Performance Graphs
Instead of viewing raw text logs inside your terminal, you can direct k6 to stand up an interactive local metrics monitor. Run the following command to initiate execution and spin up the live visual dashboard interface:
K6_WEB_DASHBOARD=true k6 run api_bench.js
Once the execution loop fires, launch a web browser window and open http://localhost:5665. The interface will plot data points continuously for concurrent threads, data transfer metrics, and end-to-end response times.
4. Compiling a Shareable HTML Graphical Report
For offline reviews or continuous integration pipelines, embed the diagnostic visualizations directly into an autonomous standalone asset. Pass the report destination using the export flag command below:
K6_WEB_DASHBOARD=true K6_WEB_DASHBOARD_EXPORT=api_performance_report.html k6 run api_bench.js
This compiles an HTML sheet bundled with chart libraries, meaning you can distribute it to cross-functional stakeholders who can zoom and slide through timelines without external dependencies.
